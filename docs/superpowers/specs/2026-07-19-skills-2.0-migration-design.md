**Date:** 2026-07-19
**Sub-project:** A of 5 (A: Skills 2.0 · B: Workspace/RAG · C: Tooling integrations · D: Lifecycle rituals · E: Operational rules)

## Goal

Migrate ABSOLUTE-CLAUDE's first-party skills from plain-markdown SKILL.md files to a "Skills 2.0" format: each skill carries a structured YAML config and a JSON Schema alongside its SKILL.md. Simultaneously prune skills made obsolete by the upcoming RAG-driven architecture (sub-projects B–D) and consolidate overlapping design skills into a single UI skill.

## Skills 2.0 format

Each skill lives in `skills/<name>/`:

```
skills/<name>/
├── SKILL.md      # unchanged role: frontmatter (name, description) for Claude Code
│                 # discovery + prose body; gains one closing line pointing at sidecars
├── skill.yaml    # structured config
└── schema.json   # JSON Schema draft 2020-12
```

Claude Code only enforces SKILL.md frontmatter; the sidecars are a repo convention the model reads on invocation and that future tooling (RAG indexing in sub-project B, validators, generators) consumes as machine-readable artifacts. This is a deliberate soft contract.

### `skill.yaml` fields

```yaml
name: <kebab-case>        # MUST equal the directory name and SKILL.md frontmatter name
version: 2.0.0
kind: knowledge | tool    # tool = ships executable scripts
triggers: [ ... ]         # activation phrases (extracted from the current description)
commands: [ ... ]         # slash commands exposed, if any
scripts: [ ... ]          # bundled executables, if any
dependencies: [ ... ]     # formalizes former [[wiki-link]] cross-references
schema: ./schema.json
```

### `schema.json` shape

Top-level `definitions` with two entries:

- `inputs` — typed parameters the skill accepts. Knowledge skills: optional free-form `context` string. Tool skills: real typed flags.
- `outputs` — expected result shape, described declaratively (e.g. "commit subject ≤ 50 chars, Conventional Commits 1.0.0 type prefix").

### SKILL.md addition

One line appended after the frontmatter body: `Config: skill.yaml · Schema: schema.json` so the model loads sidecars when the skill is invoked.

## Validator

`skills/validate-skills.mjs` — dependency-free Node (repo convention: `node:` builtins only):

- For every directory under `skills/`: SKILL.md exists and frontmatter parses; `name` matches directory name and `skill.yaml.name`; `skill.yaml` parses with required fields (name, version, kind, triggers, schema); `schema.json` is valid JSON containing `definitions.inputs` and `definitions.outputs`; every entry in `scripts` exists on disk.
- Exit 1 with an error listing on any failure; exit 0 otherwise.
- Runnable standalone and from the installer (`install.sh --verify` or as part of the summary step).

## Model policy (already applied)

Appended to `~/.claude/CLAUDE.md`:

- Agent-tool spawns for subagent-driven development must pass `model: "sonnet"` (claude-sonnet-5).
- Code reviews are never delegated to subagents; they run in the main session on the current model (typically Fable 5 / Opus 4.8).
